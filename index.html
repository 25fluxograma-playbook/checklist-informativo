Checklist - Informações Comprometidas

Este guia fornece uma estrutura técnica para resposta a incidentes de segurança. Siga os passos sequencialmente para garantir a contenção, análise, erradicação e recuperação total do ambiente afetado.

1. CONTENÇÃO E PRESERVAÇÃO

1.1 Houve alerta de comprometimento?
1.2 O sistema foi isolado logicamente de forma imediata?
1.3 A memória RAM foi capturada?
1.4 As sessões ativas foram registradas?
1.5 As conexões estão sendo monitoradas?

2. ANÁLISE

2.1 A criticidade do incidente foi avaliada?
2.2 Existe risco de propagação imediata?
2.3 O responsável pelo servidor foi contatado?
2.4 O custo-benefício de manter o servidor online foi avaliado?
2.5 A decisão final sobre o estado do servidor foi tomada?
2.6 O servidor foi isolado totalmente (offline)?
2.7 O monitoramento foi reforçado?

Tipo de Informação Comprometida:

Credenciais de acesso?
Dados pessoais/sensíveis?
Informações financeiras?

3. ERRADICAÇÃO E RECUPERAÇÃO

3.1 O malware ou vetor foi identificado?
3.2 O malware foi removido?
3.3 As vulnerabilidades foram corrigidas?
3.4 Backdoors foram verificados e eliminados?
3.5 Os sistemas foram restaurados?

4. RECUPERAÇÃO E CONFORMIDADE

4.1 A reintegração foi feita de forma gradual?
4.2 A integridade dos sistemas foi verificada?
4.3 O incidente foi documentado?
4.4 A conformidade foi verificada?
4.5 Notificações regulatórias foram enviadas (se aplicável)?

5. ATIVIDADES PÓS-INCIDENTE

5.1 Foi realizada análise da causa raiz?
5.2 As lições aprendidas foram registradas?
5.3 As políticas foram atualizadas?
5.4 A equipe recebeu treinamento?
5.5 Houve ações de melhoria contínua?
